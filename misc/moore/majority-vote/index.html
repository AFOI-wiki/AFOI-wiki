    摩尔投票法¶

    by Acc_Robin

    tip

    如果你觉得本文比较突兀，可以先看看原 OI-wiki 中的主元素问题

    简述¶

    在 空间， 时间求解序列主元素（主元素指出现次数超过 的元素）。

    详解¶

    不断消去两个相邻的、不同的元素之后，剩下的一定是主元素。

    需要支持的操作¶

    1. 栈
    2. 判断两元素是否相同。

    可合并性¶

    摩尔投票法支持合并：合并后的主元素要么不存在，要么一定是合并前两个集合主元素之一。

    int moore(const vector<int>& vec) {
    int v = -1, c = 0;
    for (int x : vec) {
        if (x == v) ++c;
        else if (--c <= 0) v = x, c = 1;
    }
    return v;
}

    线段树维护摩尔投票¶

    请注意区分主元素与众数的关系：主元素要求出现次数严格大于区间长度的一半（区间众数规约矩阵乘法）。

    线段树每个节点存储一个对子 ， 表示可能是主元素的值，而 就是摩尔投票相消去之后剩下的次数。

    此处“可能”指的是：不是 的值一定不是主元素，但 不一定是主元素（有可能不存在主元素）

    合并时如果两区间 相同，则 相加；否则大区间的 为 更大的那个区间的 ， 为两区间 的差。

    struct T{
    int v, c;
};
auto mg=[](T a, T b)->T {
    if (a.v == b.v) return {a.v, a.c + b.c};
    return {a.c > b.c ? a.v : b.v, abs(a.c - b.c)};
};

    这样维护只能保证：若区间存在主元素，则 一定是主元素，但如果区间不存在主元素， 就不能保证了。

    因此，应当在查询完区间 之后，使用其它数据结构判断 在区间出现次数是否严格大于区间长度的一半。通常可以对每个值开一颗平衡树，存储这个值出现的所有下标即可。

    P3765 总统选举：线段树维护摩尔投票。
